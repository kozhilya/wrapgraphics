\begin{abstract}
  \texttt{wrapgraphics} is a Lua\LaTeX{} package that wraps text around an
  image following the contour of its alpha channel. A Python backend
  (Pillow) traces the contour; Lua computes \mintinline{latex}|\parshape| entries.
\end{abstract}

\section{Introduction}

\texttt{wrapgraphics} is a Lua\LaTeX{} package that wraps text around an
image following the contour of its alpha channel.

\begin{minted}{latex}
\usepackage{wrapgraphics}
...
\wrapgraphics[threshold=0.5, padding=5, scale=0.20]{image.png}
\end{minted}

The text following \texttt{\string\wrapgraphics} will flow around the image.
The image is placed as an overlay so it sits inside the paragraph without
breaking the text flow.

The approach used in this package is similar to the approach in the
\href{https://ctan.org/pkg/minted}{\texttt{minted}} package:

\subsection{How it works}

\begin{enumerate}
  \item \mintinline{latex}|\wrapgraphics| saves the image in a box, then
        calls Lua via \texttt{\string\directlua}.
  \item Lua determines the contour SVG path and calls
        \texttt{wrapgraphics.py} if a cached SVG
        with matching parameters does not already exist.
  \item Python extracts the alpha channel, applies threshold and
        dilation (padding), traces the outer contour with the
        Moore--Neighbor boundary following algorithm, and writes a
        \texttt{-shape.svg} file.
  \item Lua parses the SVG, converts pixel coordinates to
        (\texttt{pt} / \texttt{sp}), and builds a flat list of
        \mintinline{latex}|\parshape| entries (indent + width per line).
  \item The \mintinline{latex}|\parshape| is injected into the TeX stream,
        and the image is placed as an overlay (\mintinline{latex}|\rlap|)
        so it sits inside the reflowed paragraph without consuming space.
  \item A \texttt{post\_linebreak\_filter} callback tracks how many
        lines have been consumed and clears the shape when the image height
        is covered, ensuring correct behaviour across page breaks.
\end{enumerate}


\subsection{Special note about AI}
AI agents (namely, Deepseek V4 Flash Free wia OpenCode) were used
to create this package, but later its codebase and examples were manually
checked by the author so that the package can be maintained without extra tools.

The author understands that the approach of this package may be sub-optimal: the
contour could probably be calculated with Lua, or the
\mintinline{latex}|\parshape| could be placed with expl3 or with Lua.

But while the author can maintain Python and some Lua produced by AI, he cannot
do the same for complex Lua (such as contour calculations) and expl3 code.
If someone else can help to produce the necessary changes to the repository, the author
will be grateful, but until then the over-complication of the package will
probably persist.

\section{Installation}

Place \texttt{wrapgraphics.sty}, \texttt{wrapgraphics.lua}, and
\texttt{wrapgraphics.py} in the same directory as your document
(or in \texttt{\$TEXMF/tex/latex/wrapgraphics/} for system-wide install).

\begin{itemize}
  \item Lua\LaTeX{} (with \texttt{--shell-escape})
  \item Python~3 + Pillow
\end{itemize}

\section{Package options}

A document that uses the \texttt{wrapgraphics} package should contain a line like the
following near the top of the document:

\begin{minted}{latex}
\usepackage[]{wrapgraphics}
\end{minted}

Supported options are \texttt{verbose} (\ref{package:verbose}) and \texttt{contour} (\ref{package:contour}).

\subsection{\texttt{verbose}}\label{package:verbose}

When enabled, additional diagnostic messages are written to the
terminal and log file.  It can be used for debugging or performance
analysis.  It should be disabled when compiling large documents.

\subsection{\texttt{contour}}\label{package:contour}

The \texttt{contour} option draws the traced contour in red over the
image for debugging and visualisation purposes.

This option can be enabled for each image separately.

\section{Macro options}

The \mintinline{latex}|\wrapgraphics| command accepts the options listed below,
passed as a comma-separated key–value list.

\subsection{\texttt{position} (default: \texttt{left})}\label{macro:position}

Which side of the image the text wraps around.  Accepted values are
\texttt{left}, \texttt{right}, and \texttt{middle}.
With \texttt{middle} the image is centred and text flows around it.

\begin{minipage}{\textwidth}
  \centering
  \preview[page=1,width=\previewwidth]{comparison-left.pdf}
  \preview[page=1,width=\previewwidth]{comparison-right.pdf}
  \captionof{figure}{Left (default) and right wrapping.}
\end{minipage}


\subsection{\texttt{threshold} (default: \texttt{0.5})}\label{macro:threshold}

Alpha threshold (0--1).  Pixels with alpha greater than the threshold
are considered opaque; the contour runs along the boundary of opaque
pixels.  A low value treats more pixels as opaque, keeping the contour
closer to the visible image edge.  A high value lets text come closer
to the core of the image.

\begin{minipage}{\textwidth}
  \centering
  \preview[page=1,width=\previewwidth]{params-threshold-03.pdf}
  \preview[page=1,width=\previewwidth]{params-threshold-08.pdf}
  \captionof{figure}{Threshold 0.3 (left) and 0.8 (right).}
\end{minipage}


\subsection{\texttt{padding} (default: \texttt{5})}\label{macro:padding}

Extra clearance in pixels between the image edge and the wrapped text.
It is implemented by dilating the alpha mask before contour tracing.

Note that value of this option is pixels of image. Because of it, same
value will produce different result for small and big images.

\begin{minipage}{\textwidth}
  \centering
  \preview[page=1,width=\previewwidth]{params-padding-50.pdf}
  \preview[page=1,width=\previewwidth]{params-padding-00.pdf}
  \captionof{figure}{Padding 50\,px (left) and 0\,px (right).}
\end{minipage}


\subsection{\texttt{scale} (default: \texttt{1})}\label{macro:scale}

Scale factor passed to \mintinline{latex}|\includegraphics|.  Overridden
by \texttt{width} if both are specified.

\begin{minipage}{\textwidth}
  \centering
  \preview[page=1,width=\previewwidth]{basic.pdf}
  \captionof{figure}{Default left wrapping with \texttt{scale=0.20}.}
\end{minipage}


\subsection{\texttt{width}}\label{macro:width}

Explicit width passed to \mintinline{latex}|\includegraphics|,
e.g.\ \texttt{width=8cm}.  Overrides \texttt{scale}.

\begin{minipage}{\textwidth}
  \centering
  \preview[page=1,width=\previewwidth]{width.pdf}
  \captionof{figure}{Explicit width (\texttt{width=8cm}).}
\end{minipage}


\subsection{\texttt{hang} (default: \texttt{false})}\label{macro:hang}

When set to \texttt{true} the indent becomes constant for all lines,
equal to the widest contour point.  Text is pushed by a fixed amount
rather than following the exact shape.

\begin{minipage}{\textwidth}
  \centering
  \preview[page=1,width=\previewwidth]{comparison-left-hang.pdf}
  \preview[page=1,width=\previewwidth]{comparison-right-hang.pdf}
  \captionof{figure}{Hang mode (left and right).}
\end{minipage}


\subsection{\texttt{lines}}\label{macro:lines}

Limit wrapping to the first $N$ lines; subsequent lines use the full
text width.

\begin{minipage}{\textwidth}
  \centering
  \preview[page=1,width=\previewwidth]{lines.pdf}
  \captionof{figure}{Wrapping limited to 5 lines.}
\end{minipage}


\subsection{\texttt{shift\_x}, \texttt{shift\_y}}\label{macro:shift}

Horizontal and vertical offset of the image
(e.g.\ \texttt{shift\_x=20pt}, \texttt{shift\_y=10pt}).
Positive \texttt{shift\_x} moves the image right;
positive \texttt{shift\_y} moves the image up.

Shifting does not affect text layout above the image;
it is most useful when the upper part of the image is transparent.

\begin{minipage}{\textwidth}
  \centering
  \preview[page=1,width=\previewwidth]{shift.pdf}
  \captionof{figure}{Image shifted right by 20\,pt and up by 10\,pt.}
\end{minipage}


\subsection{\texttt{anchor} (default: \texttt{here})}\label{macro:anchor}

Where to anchor the image.  The value \texttt{here} (default) places it
inline at the paragraph start.  The compass-point values
(\texttt{nw}, \texttt{ne}, \texttt{sw}, \texttt{se}) pin it to the
corresponding corner of the text area (north-west = top-left, etc.).

\begin{minipage}{\textwidth}
  \centering
  \preview[page=1,width=\previewwidth]{anchor.pdf}
  \captionof{figure}{Image anchored to the top-right corner (\texttt{anchor=ne}).}
\end{minipage}


\section{Examples}

\subsection{Multiple images}

Multiple \texttt{\string\wrapgraphics} calls can be placed throughout the
document.  Each call independently traces the contour and generates the
appropriate \texttt{\string\parshape}.

\subsection{Large image sticking out}

A large image (wider than \texttt{\string\textwidth}) makes the wrapping
indentation exceed the text width; the right side of the image protrudes
into the margin.
