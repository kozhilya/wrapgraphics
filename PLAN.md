# Plan — wrapgraphics

## Architecture

- **Python** reads image alpha channel, traces contour at given threshold, pads by N pixels, writes contour to `{image}-shape.svg` (image + red path overlay)
- **LuaLaTeX** (`.sty`) calls Python via `os.execute`, reads SVG with string matching, computes per-line margins, applies them via `\parshape`, places the image

## Data flow

```
.tex ──\wrapgraphics──► .sty ──\write18──► wrapgraphics.py
                                                 │
                                  image.png ─────┤
                                                 │
                                  threshold/pad ─┤
                                                 │
                                                 ▼
                                  image.png-shape.svg
                                                  │
               .sty ◄─── \directlua{gmatch()} ◄───┘
              │     (compute margins, emit \parshape)
              ▼
            PDF with wrapped text + image
```

## File output format

`image.png-shape.svg` is an SVG file viewable in any browser:

```svg
<svg xmlns="http://www.w3.org/2000/svg" width="800" height="600"
     wg-dpi="72.0" wg-threshold="0.5" wg-padding="5">
  <image href="image.png" width="800" height="600"/>
  <path d="M 10.0 20.0 L 15.0 25.0 ... Z" fill="none" stroke="#f00" stroke-width="2"/>
</svg>
```

- The `<image>` embeds the source picture so users can see the contour overlaid
- The `<path>` draws the computed (simplified) contour in red
- The `wg-dpi`, `wg-threshold`, `wg-padding` attributes carry metadata
- Points are `{x, y}` in image-pixel coordinates (x = column, y = row from top)

## Key decisions

| Decision | Choice |
|---|---|
| LaTeX engine | LuaLaTeX only (native Lua for margin computation) |
| Wrapping mechanism | Lua parses SVG contour, computes \parshape at compile time |
| Python dependencies | Pillow only (minimum deps) |
| Contour tracing | Moore-Neighbor boundary following (pure Python) |
| Documentation language | English |
| Demo/test image | Generated by Python script (circle with alpha) |

## Granular task list

### Phase 1 — Python CLI (`wrapgraphics.py`)

- [x] 1.1 Read image, convert to RGBA, extract alpha channel as `L`-mode image
- [x] 1.2 Threshold alpha at given level (0–1): values ≥ level → opaque
- [x] 1.3 Dilate binary mask by N pixels via `MaxFilter` (padding clearance)
- [x] 1.4 Find starting opaque pixel (top-left scan)
- [x] 1.5 Trace outer contour with Moore-Neighbor boundary following
- [x] 1.6 Simplify contour with Ramer-Douglas-Peucker (epsilon=1.0)
- [x] 1.7 Write contour + image metadata to `{image}-shape.svg` file
- [x] 1.8 CLI with `--input`, `--output`, `--threshold`, `--padding` args

**Refinements (future):**

- [x] 1.9 Handle fully transparent images: return empty contour, LaTeX shows no indent
- [x] 1.10 Handle images without alpha channel (non-RGBA): treat as fully opaque
- [x] 1.11 Add `--simplify` / `--no-simplify` flag to control RDP
- [x] 1.12 Add `--epsilon` parameter for RDP tolerance
- [x] 1.13 Add `--invert` flag (wrap on left side instead of right)
- [x] 1.14 Add verbose/logging mode (`--verbose`)
- [x] 1.15 Handle errors gracefully (corrupt images, permission errors, etc.)
- [ ] 1.16 Support multiple contours (separate connected components) → each produces its own contour table
