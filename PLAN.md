# Plan — wrapgraphics

## Architecture

- **Python** reads image alpha channel, traces contour at given threshold, pads by N pixels, writes contour data to a Lua-source file
- **LuaLaTeX** (`.sty`) calls Python via `os.execute`, reads contour data with `dofile`, computes per-line margins, applies them via `\parshape`, places the image

## Data flow

```
.tex ──\wrapgraphics──► .sty ──\write18──► wrapgraphics.py
                                                 │
                                  image.png ─────┤
                                                 │
                                  threshold/pad ─┤
                                                 │
                                                 ▼
                                        image.shape.lua
                                                 │
              .sty ◄─── \directlua{dofile()} ◄───┘
              │     (compute margins, emit \parshape)
              ▼
            PDF with wrapped text + image
```

## File output format

`image.shape.lua` contains a Lua-returnable table of contour points:

```lua
return {
  {10, 20},
  {15, 25},
  ...
}
```

Each point is `{x, y}` in image-pixel coordinates (x = column, y = row from top).

## Key decisions

| Decision | Choice |
|---|---|
| LaTeX engine | LuaLaTeX only (native Lua for margin computation) |
| Wrapping mechanism | Lua parses contour, computes \parshape at compile time |
| Python dependencies | Pillow only (minimum deps) |
| Contour tracing | Moore-Neighbor boundary following (pure Python) |
| Documentation language | English |
| Demo/test image | Generated by Python script (circle with alpha) |
